# OpenManic implementation decisions

Create a short decision record here only when implementation evidence changes or narrows a durable product or architectural choice. Ordinary code decisions belong in types, tests, rustdoc, and commit history.

Each record must include:

- date and task ID;
- decision and status;
- evidence considered;
- affected requirements and contracts;
- alternatives rejected and why;
- migration, compatibility, or follow-up consequences; and
- primary-agent approval.

File names use `NNNN-short-description.md`. Do not create a decision record to retroactively justify an undocumented agent assumption.
